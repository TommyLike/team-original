# [openEuler文档写作插件上线，为开发者打造高效文档工作流](https://mp.weixin.qq.com/s/E8j7I4UWOclt-gJSeWa3Qw)

[OpenAtom openEuler](javascript:void%280%29;)*2025-11-05 18:30:00广东*

相信很多开发者都有这样的体会：技术思路清晰明了，文档撰写却繁琐枯燥。对此，OpenAtom openEuler 发布了 Doc Tools 和 DocMate 两款 VS Code 插件，旨在让你的文档工作变得更轻松、更高效。

**01**

Doc Tools：让文档检查自动化、实时化

还在为标点、死链、命名不规范…反复检查文档？耗时又耗神？

试试 Doc Tools，让你的文档写作省力又省心！

- ? 检查全自动：自动检查标点、链接、文件名等多种问题，并支持实时预览效果。

<!--THE END-->

- ?? 纠错实时化：写作即校对，问题实时发现、即时修复，告别“文档门禁”的被动等待。

**02**

DocMate：让文档从“规范”迈向“优美”

格式问题完美解决，但内容表达不够出彩？

试试 DocMate，让你的文档从“规范”迈向“优美”！

感谢开源之夏项目南京大学李思懿（@Delthin）的贡献。

- ? AI 文本润色：一键优化表达，让语言更流畅、风格更专业。

<!--THE END-->

- ? 智能内容改写：支持自定义改写指令，满足多样化的内容需求。

<!--THE END-->

- ? 智能内容检查：精准识别错别字、文档风格不一致等问题。

**03**

立即体验，开启高效写作

安装

在 VS Code 的扩展商店中安装“Doc Tools”和“DocMate”。

![](https://mmbiz.qpic.cn/mmbiz_png/jqTxnpIZ2mBG8bQ9GyTuMBWgLpehDAxTnh12edu3TKpAC1xKxQp9QCnHcV0yyLRWJb7nDrDVzKA8MyQyccjlBQ/640?wx_fmt=png&from=appmsg)

使用

Doc Tools 使用指导：https://docs.openeuler.openatom.cn/zh/docs/common/contribute/doc\_tools\_introduction.html

DocMate 使用指导：https://docs.openeuler.openatom.cn/zh/docs/common/contribute/doc\_mate.html

反馈与建议

欢迎在gitee提交 Issue，助力我们做得更好。

https://gitee.com/openeuler/docmate/issues 

了解更多

可查阅《文档开发指南》

https://docs.openeuler.openatom.cn/zh/docs/common/contribute/contribution\_process.html

此外，我们今年优化了开源文档的生产、发布机制，文档生产下沉到各 SIG 组，各 SIG 组协同分工。此机制已在 openEuler 25.09 版本全面落地，覆盖20余个 SIG 组及全部80余本手册。

![](https://mmbiz.qpic.cn/mmbiz_png/jqTxnpIZ2mCyuWGvMWkuCybXicTbDlR307iclQVdjFjp6tXjrBUrgHu4y6NFfMxiaf1BwkOo8snT00eCqib5tJ1vcA/640?wx_fmt=png)

同时，我们按业务场景，分用户角色对产品文档架构进行了重构，将文档拆分成6大业务场景，8个工具模块，并优化了文档中心视觉效果、交互方式、搜索体验、评分机制等。
