# [openEuler MIC Meetup 成功举办,谁说不写代码就不能玩开源](https://mp.weixin.qq.com/s/eYV4ay4zdDz1F4njebgDAA)

[OpenAtom openEuler](javascript:void%280%29;)*2021-03-22 19:03:00*

3月20日，由 openEuler 社区联合蘑菇云创客空间举办的 openEuler MIC Meetup 上海站成功举办。本次活动汇聚行业领袖、业界资深专家、社区技术委员、Maintainer、开发者及用户，围绕开源项目展开深入探讨。

开源项目健康运作离不开度量，openEuler 联合 Linux 基金会顶级项目 CHAOSS，共同探讨社区健康度量体系。

![](https://mmbiz.qpic.cn/mmbiz_jpg/A0h5yD51CMaNMTQDmtPTDLc295nNCu1Mn5GAUia9krrze26XYT0qO76Uhdgvu2ufiaMd98icKJktW0YrdyTvC1JuA/640?wx_fmt=jpeg)

Elizabeth Barron & Sean Goggins 两位讲师通过视频介绍 CHAOSS 社区度量体系。

现场首先由来自华东师范大学的夏小雅老师分享 2020 GitHub 数字洞察。

紧接着由来自上海交通大学的教授曹健和华为工程师李自共同分享开源社区大数据分析：从度量到洞察。

随后由 openEuler infra SIG Maintainer 钟君分享议题“ openEuler 社区运营看板”。现场介绍 openEuler 的数字运营平台，讲述如何构建自动化数据采集及处理平台。

当前的开源运营数据指标系，这些指标体系该如何指导运营，George.Cao 的议题“利用开源构建 openEuler 基础设施”，介绍了 openEuler 社区基础设施架构，现状，后续规划。

![](https://mmbiz.qpic.cn/mmbiz_jpg/A0h5yD51CMaNMTQDmtPTDLc295nNCu1Myr7KictFVvfPTv24QnmFhMjJach6xb7zAWN0icMtOESCnrYo9icugpSDw/640?wx_fmt=jpeg)

钟君分享议题

开发者是改变世界的力量，通过开发者所贡献的一行行代码，构建起生机勃勃的开源世界，如何给开源开发者带来更好的体验，是体验团队需要一直探索实践的。

开源社区体验设计师张舒婷主要讲述了他们在 openEuler 开源社区，从开发者体验出发，作出的设计和运营实践。

openEuler 技术委员会委员马全一和工程师陈曾与大家分享议题“开源 CLA 解密”。让大家了解什么是 CLA，为什么要签署 CLA，应该签哪种 CLA。具体讲述代码贡献者签署 CLA 注意事项，签署企业 CLA 的流程。

杨聪

开源软件合规（Compliance）中企业使用开源软件许可证（License）的合规是基础和重点，而 License 合规知识在开发团队中的普及，以及合规检查的工具化、自动化，又是其在开源合规实践中遇到的重点和难点。

现场介绍了 openEuler 在开源合规中的初步探索和实践， 他们建立了一个开放共享的 License 合规门户网站和合规信息数据库，提供持续更新的 License 合规信息；同时，利用该合规信息数据库，开发了 openEuler 的 License 合规检查门禁工具。对社区开发人员每次提交的 PR，提供 License 合规报告。

最后，中科院软件所科技成果转化项目负责人杨牧天与大家分享开源许可证识别以及许可证条款冲突自动化检测，呼吁大家一同“护开源生态，筑安全未来”。

![](https://mmbiz.qpic.cn/mmbiz_jpg/A0h5yD51CMaNMTQDmtPTDLc295nNCu1MX8ZTibmVMEGC1aXdtqmMsJsFFmPNztSicMYHeKmDyBBwzBSpiaNVobURA/640?wx_fmt=jpeg)

提问环节

合规是开源项目的底线，往后 openEuler 会继续在开源软件领域推广合规和许可证的理念，帮助开发者更好地使用开源软件，防止在商业化的过程中遇到法务风险。openEuler 未来也会继续更多数字化工具，帮助开发者更合理地商业化使用开源软件，减少在商业活动中的法务风险。蘑菇云创客空间也会继续支持 openEuler 的活动，更好地帮助创客和创业者商业化项目。
