SIG 是 Special Interest Group 的缩写，中文翻译为特别兴趣小组，openEuler 社区是按照不同的 SIG 来进行组织的，以便于更好的管理和改善工作流程。

openEuler SIG 一般有以下几个特点：

- SIG 组和 SIG 的邮件列表是开放的，欢迎任何人和团体加入并参与贡献。
- SIG 都是针对特定的一个或多个技术主题而成立的。SIG 内的成员推动开源项目输出，并争取让开源项目成为 openEuler 社区发行的一部分。
- SIG 的核心成员主导 SIG 的治理。请查看SIG 的角色说明\[1]。您可以在贡献的同时积累经验和提升影响力。
- 每一个 SIG 在 Gitee 上都会拥有一个或多个项目，这些项目会拥有一个或多个 Repository。SIG 的开源项目会保存在这些 Repository 内。
- 可以在 SIG 对应的 Repository 内提交 Issue、针对特定问题参与讨论，提交和解决问题，参与评审等。
- 您也可以通过邮件列表、IRC 或视频会议和 SIG 内的成员进行交流。

## 保姆级教程：

首先，打开 SIG 列表\[2] 查看所有 SIG。

如果有，那么请直接加入。可以直接在 SIG 列表所在的 community 仓库提交即可。例如：go\\-sig：add new sig golang · Pull Request \\!398 \[3] 。

注意，加入 SIG 组不需要必须是 committer 或者 maintainer。SIG 组中成为 maintainer&committer 首先要有稳定的贡献率，所以您可以先从 contributor 做起，当您做了足够贡献，SIG maintainer 会发起投票，然后您才可以成为 committer。

如果您想加入的 SIG 不存在，那么您需要按照模板提交 PR 到 community 仓库，建立新的 SIG 组模板\[4]在这。

在您提交请建立新 SIG 的 PR 后，记得订阅 openEuler 技术委员会（简称 TC）邮件列表\[5] ，方便接收 TC 社区工作会议的邮件通知。TC 在社区工作会议上对申请进行评审。

![](https://mmbiz.qpic.cn/mmbiz_gif/A0h5yD51CMZVIM2LlrJWrwdiab3ia6icsyBu7dgCpB5ZAWoSibg6hL2WdgMtQpZXWHYtPwKym2hOf6NQK5Brdv7m8w/640?wx_fmt=gif)

订阅邮件列表的步骤

如果评审通过，则将您提交的 PR 合入到 community 仓库，也就意味着申请的 SIG 组被批准了。

如果评审不通过，那么您提交的 PR 将会被拒绝合入到仓库，需要根据 openEuler 技术委员会的意见进行修改。

如果您通过上述以上两种方式都找不到感兴趣的 SIG，您可以向 community@openeuler.org 发求助邮件，建议您使用“开发过程疑问”作为标题，在内容中写出您寻找的 SIG 或项目特征，openEuler 社区会为您提供帮助。

openEuler Summit 将 TC 工作会议从线上搬到了线下，这其中就包括了对新 SIG 的评审环节。**你可以现场提交 SIG 申请，TC 主席和委员在现场工作会议中对新 SIG 申请进行评审**。

此外，我们在现场给 20个 SIG 预留了现场工作会议的场地，一度的“网友”终于可以在现实世界中一起开会，对于 openEuler 下一个版本的技术路线和发布目标。

不要迟疑了，长按二维码提交申请或议题吧！
