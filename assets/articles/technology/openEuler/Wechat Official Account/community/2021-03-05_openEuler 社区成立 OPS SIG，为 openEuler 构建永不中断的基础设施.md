# [openEuler 社区成立 OPS SIG，为 openEuler 构建永不中断的基础设施](https://mp.weixin.qq.com/s/uo8HVoESqS1IZfu4KSZH7g)

[OpenAtom openEuler](javascript:void%280%29;)*2021-03-05 19:11:09*

经openEuler 社区技术委员会讨论决定，openEuler 社区正式成立 OPS SIG。该 SIG 主要在 openEuler 社区推动构建系统运维的可视化、自动化、智能化的基础设施；构建系统数据体系，使数据能跨组件，跨平台共享；提供基础设施热补丁、热替换软件技术，保证基础设施服务不中断。

 

OPS SIG 得到了华为、中国移动、招商银行、统信软件、复旦大学、中科院软件所的支持。相关工程师将作为 OPS SIG 的 Maintainer。

基于openEuler的商业发行版越来越多，如何保证使用商业发行版的用户在使用 openEuler 的过程中，遇到系统问题时快速相应？运行在 openEuler 上的应用程序遇到崩溃、服务中断等问题如何收集崩溃日志？这是 OPS SIG 需要解决的问题。

 

为了更好的解决上述问题，OPS SIG 决定建立一个可视化、自动化、智能化的运维体系。通过大量的收集应用崩溃日志，与后台专家的知识模型结合，综合大数据、人工智能和深度学习算法，最终构建出 openEuler 的 AIOPS 运维体系。提高解决问题的效率。

 

让openEuler 和 openEuler 商业发行版的用户能够享受高可靠性、高性能、永不中断的系统服务。

 

为了能够实现这样的目标，需要 OPS SIG 在数据组件上实现跨平台。一方面是为了方便开发者更好的基于该组件构建上层业务，另外一方面，这是保障全栈操作系统运维非常重要的一个步骤。

 

OPS SIG 在未来将会提供问题诊断、性能分析、系统调测等技术。结合 A-OPS 调测工具，可以帮助用户更好的定位系统问题。配合相应的基础设施热补丁、热替换软件技术更好的解决问题。
