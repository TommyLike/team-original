# [openEuler 社区成立 Embedded SIG，构建 openEuler 的嵌入式操作系统](https://mp.weixin.qq.com/s/WCP4arCdWZalstO7gROV4Q)

[OpenAtom openEuler](javascript:void%280%29;)*2021-03-16 18:00:00*

经 openEuler 社区技术委员会讨论决定，openEuler 社区正式成立 Embedded SIG。Embedded SIG 致力于开发出一个可以运行在嵌入式设备上的 openEuler 操作系统。

在过去的一年里，openEuler 社区取得了长足的进展，其丰富程度早已超过了成立之初的范围。

现在，openEuler 社区已经开始从着重于服务器，云基础设施等标准的 IT 基础设施，向多场景，多领域发展。这次成立的 Embedded SIG 就是这种趋势的一种体现。

 

在操作系统领域，传统上服务器 OS 和嵌入式 OS 都是井水不犯河水的存在，但是随着端云一体趋势的发展，随着嵌入式设备运算能力的进一步增强，边缘计算的兴起，标准的 OS 和嵌入式 OS 的边界在渐渐模糊。

那么是否存在一种可能，让我们在同一个 OS 基础上，同一个软件基线基础上，同时能够构建出服务器&云的 OS，边缘计算的 OS，嵌入式的 OS。

相信这将是一个非常有趣的探索方向。现在，我们终于有了一个新的 Embedded SIG 来承接这个挑战。

 

Embedded SIG 致力于开发出一个可以运行在嵌入式设备上的 openEuler版本。

开发者后续可以使用 Embedded SIG 提供的镜像将 openEuler 安装到嵌入式硬件上，或者使用 Embedded SIG 提供的工具，量身定制出适合自己硬件的 openEuler 操作系统。

 

嵌入式场景是类 Linux 操作系统的重要使用场景，从智能手机到路由器，再到智能网关，都有 Linux 的身影。openEuler 作为一个开放的操作系统社区，完全支持Embedded SIG 将 openEuler 能够应用到越来越多的设备和场景中。

Embedded SIG 的主要技术目标有以下几点：

 

第一：在 openEuler 社区中添加对主流开发板以及嵌入式设备的支持；

第二：在 openEuler 社区中添加嵌入式领域相关开源软件的支持；

第三：实现 openEuler 内核和软件包的轻量化，并实现系统裁剪系统开发；

第四：upstream first，将 openEuler 在嵌入式上的成果回馈给上游社区;

第五：积极听取 openEuler 社区中对于嵌入式设备的用户反馈，解决用户遇到的问题。
