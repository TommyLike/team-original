# [openEuler 社区成立 Cloud Native SIG](https://mp.weixin.qq.com/s/iRaIbgj74kFYv-j0STNSDw)

原创*周荔人*[OpenAtom openEuler](javascript:void%280%29;)*2021-02-03 19:10:00*

经 openEuler 社区技术委员会讨论决定，openEuler 社区正式成立 Cloud Native SIG。Cloud Native SIG 的成立将进一步完善和丰富云原生基础设施对 openEuler 操作系统的支持，openEuler 社区的贡献者和用户可使用由 Cloud Native SIG 维护的云原生基础设施，并且通过 openEuler 社区获得技术支持。

![](https://mmbiz.qpic.cn/mmbiz_jpg/A0h5yD51CMa7lLCBqDyB4o1cXb6YL68T5lz5mbpYpHnXmazLPsQ0HuwqSP1RRbPEFrDF3WYRKSx1xukQWpqmwQ/640?wx_fmt=jpeg)

 

Cloud Native SIG 组成立不仅显示了社区对于云原生领域的重视，同时也聚集了一批来自社区不同维度的贡献者，共同参与云原生领域的创新与探索，希望引领 openEuler 社区技术领导力。当前 Cloud Native SIG 的 Maintainer 来自于中国移动、中科院软件所、麒麟软件、润和软件、华为、青云。

 

云原生是云计算时代的技术理念，有着传统 IT 无法比拟的优势，可以帮助企业高效享受云的弹性和灵活性，从而实现平滑迁移、快速开发、稳定运维，降低技术成本，云原生已经成为企业数字化转型的基础与技术趋势。openEuler 作为最活跃的开源社区，在技术上会持续保持业界领先，成立 Cloud Native SIG 是这种技术领先性的一种体现。

 

本次 Cloud Native SIG 的成立也是 openEuler 社区 SIG 组的自我优化。在 Cloud Native SIG 没有成立之前，云原生技术分散在各个 SIG 组之间，例如：Container SIG、Kubernetes SIG、OKD SIG、iSulad SIG。各个 SIG 组在云原生技术上分而治之，在职能上有所重复。openEuler 社区缺少一个统一的云原生基础设施组织，来帮助这些 SIG 在云原生基础设施平台上保持统一的技术标准。

 

所以 Cloud Native SIG 成立了。Cloud Native SIG 合并了 Container SIG 和 Kubernetes SIG。OKD SIG 和 iSulad SIG 的部分软件包移交给了 Cloud Native SIG 进行管理。

 

Kubernetes SIG 将改名为 KubeSphere SIG，持续在 openEuler 上为用户提供面向云原生应用的分布式操作系统 KubeSphere。

Cloud Native SIG 成立后，Container SIG、OKD SIG、KubeSphere SIG 和 iSulad SIG  将会专注于云原生应用层面的开发和适配。

![](https://mmbiz.qpic.cn/mmbiz_jpg/A0h5yD51CMa3FCrvibnyVjKPMeB9wv6SDIAVEfpx9P1atStmA8Gur6Z7hllA2vmWC7ufROibf2STHr6Mbcq8N0wg/640?wx_fmt=jpeg)

新成立的 Cloud Native SIG 将会专注于 openEuler 社区云原生基础设施的构建，为其他云原生组件提供便捷、易用的云原生基础设施和应用开发托管环境。同时持续在 openEuler 社区上进行云原生领域的技术创新、孵化新的云原生项目。

 

**Cloud Native SIG Roadmap**

 

**2021.01**

SIG 成立

**2021.06.30**

自动化安装集；容器镜像仓库；轻量级虚拟化 StratoVirt 使能；iSulad shimV2 上线

**2021.09.30**

OLM/Operator/HelmHub上线；容器 OS；机密容器；应用沙箱WASM；高性能容器存储；

**2021.12.30**

isula-build 支持 OperatorFramework/SDK；Volcano调度算法增强；StratoVirt 标准虚机

**2022.3.30**

云原生应用市场；云原生应用开发套件；A-OPS 服务化；openDonau；高性能数据面框架；EBPF网络加速
