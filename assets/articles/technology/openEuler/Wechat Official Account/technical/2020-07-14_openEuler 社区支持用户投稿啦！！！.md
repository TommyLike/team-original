# [openEuler 社区支持用户投稿啦！！！](https://mp.weixin.qq.com/s/wbMZZtQSMXwddtFWvnj0PQ)

原创*周荔人*[OpenAtom openEuler](javascript:void%280%29;)*2020-07-14 21:00:08*

openEuler 是一个开源、免费的 Linux 发行版平台，将通过开放的社区形式与全球的开发者共同构建一个开放、多元和架构包容的软件生态体系。通过软件全堆栈的技术优化，充分释放多样化计算平台算力。同时，openEuler 也是一个创新的平台，鼓励任何人在该平台上提出新想法、开拓新思路、实践新方案。

如果你愿意发表一些对于 openEuler 看法见解，那么我们非常欢迎，我们很乐意帮助你，将你的想法向 openEuler 社区中的小伙伴分享。

**投稿方向**

如果你对这项活动充满热情，但却无从下笔，我们也准备了一些写作方向，希望能够帮助到你。

**贡献软件包**：openEuler 还是一个不断完善的系统，如果你在使用 openEuler 的过程中，发现它缺少一些软件包，并且愿意将这个软件包补上，那么我们很乐意将这个软件包和您的文采，一并展示给 openEuler 的小伙伴。

**开发新软件**：如果你愿意基于 openEuler 开发一些新软件，并且将软件贡献到 openEuler 社区，那么请记得投稿。我们可以让你开发的软件，在 openEuler 社区人尽皆知。

**系统新特性**：openEuler 是一个不断完善的系统，新特性的会让系统更加完善，同时让系统充满新的可能性。希望你能通过你的文笔，将这种可能性展现在读者面前。

**版本更新**：每一次的系统更新都会发布一些新特性，告诉大家系统更新了什么，需要很多词语修饰，只是告诉大家更新了哪些东西就好了。

**SIG 宣传**：欢迎在 openEuler 社区成立各种 SIG。请准备好 SIG 的宣传材料，openEuler 公众号愿意为 SIG 提供宣传推广渠道，希望各种 SIG 能够通过这个渠道，找到更多志趣相投的爱好者，我们也欢迎 SIG 将项目中的重要进展同步到 openEuler 公众号。

以上是的建议的写作方向，如果你有其他的写作方向，也欢迎你来投稿。我们对于稿件能够达到的目的只有一个：能够繁荣 openEuler 的软件生态。

**投稿要求**

我们希望你的投稿在文章结构上是清晰的，在用词上是准确的，这是投稿文章基本的要求。

工程师是一个充满创造力群体，在公众号上的文章也不是项目的 README。所以如果你愿意的话，你也可以在行文之间添加一些诙谐幽默的元素，让你的文章不但有料而且有趣，更易于在社区的小伙伴和微信中传播。

**你可以获得的东西**

参加 openEuler 投稿活动，可以帮助你构建包括但不限于在 openEuler 社区的品牌影响力。从 openEuler 背后的那个男人，变成到台前乐于跟大家分享的知识的技术大佬。相信在不久的将来，就会有「我是看着你的文章」长大的小迷弟或小迷妹（可能会有）找你签字合影。

openEuler 还是个年轻的社区，社区中需要一些 openEuler 技术专家来为其发生发光发热。而且我知道工程师很乐于去做这样的事情。一个很现实的例子就是，在7月份进行的所有直播，其实是 openEuler 社区的工程师主动提出来的，而运营只是帮助工程师做执行而已。乐于分享是工程师文化之一，所以如果你愿意的话，可以从一篇篇文章开始，在文章中理清自己的思路，慢慢学会如何成为一个 openEuler 社区的技术专家。

openEuler 社区的相关的图书音像制品目前也在规划当中。图书的最细颗粒度当然是以文章为单位的。写作功底也需要通过一篇篇文章来进行训练。怎么说都逃不过写作这个环节，视频课程更是如此。
