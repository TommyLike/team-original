# [听说 openEuler 拿了一个很极客的奖了？](https://mp.weixin.qq.com/s/yd7L7QZXr8HYJjDpvMDj_Q)

openEuler 又双获奖了。

这次的奖的评选方式很极客，因为这是一个用算法算出来的奖。

这一奖项考察的是开源组织在 Gitee 上的活跃与项目、社区健康情况，具体从以这几个角度来衡量：**组织关注总数a、组织仓库总数b、组织下项目 Star 总数c、组织成员总数d、组织下项目 2020 年新增 fork 数e、组织下项目 2020 年合并 pr 数f、组织下项目 2020 年 issue（任务）数g。**

从前4个数据可以看到，这个考察的是项目总体的数据。后3个数据展示的是项目在 2020 年的增长数据。

从另外一个维度讲，组织成员、年度合并PR 反应的是社区开发者的参与情况，issue 反应的是用户参与情况，关注数和Star数据反应的是市场人气情况，仓库数量反应组织的开源规划情况，fork数据反馈的是项目下游的参与情况。

这些是客观的数据，是项目在日常的运作工程中一点一点积累起来的，体现了项目的活跃程度。但这还不够，真正的好项目拼的就是“综合实力”，这个“综合实力”就是权重。

通过分析评选公式：

**a\*5%+b\*15%+c\*10%+d\*25%+e\*10%+f\*25%+g\*10%** 

可以看到优秀 Gitee 组织奖看重的是项目参与的开发者数量、组织对项目规划和开发者的对项目关注程度。

openEuler 10月的运作报告中指出，openEuler 目前的 PR总和已经超过了**20000**个，平均每天有**60**个 PR 进入到社区，Kubernetes目前有 **60000**个 PR，作为一个**10**个月大的开源社区，openEuler已经达到老牌开源社区的三分之一。这点和评奖公式要体现的点是契合的，体现了openEuler是一个充满活力的开源社区。
