# [从南开、兰大欧拉操作系统创新实践课，看欧拉操作系统的机遇与挑战](https://mp.weixin.qq.com/s/16kGbdNF8aPSN1EdNqmEag)

[OpenAtom openEuler](javascript:void%280%29;)*2021-07-30 17:00:00*

7 月上旬，正是“数九三伏”的入伏时节。赶在同学们“逃离”校园开始乐享一夏前，我们决定遵循先人的教诲——夏练三伏，祭出欧拉开源社区针对高校本科生开发的欧拉操作系统开源创新实践课，而各大高校的“暑期小课堂”、“暑期实践活动”无疑是最合适的平台。

《openEuler 开源创新实践课》是计算机专业课《操作系统》的拓展课程，以欧拉操作系统为开发环境，利用 LFS（Linux from Scratch）项目从 0 到 1 构建出一个精简的 Linux 操作系统。

在这个过程中培养学生学习 Linux 的兴趣，了解 Linux 操作系统构成，提高动手实践能力，同时引导学生参与开源、贡献开源。本课程设计为 0.5 天理论授课和 1.5 天动手实践。理论授课主要介绍开源文化、Linux 发行版及欧拉开源社区；动手实践则是完成 LFS 构建并向社区提交 PR。

7 月 5 号至 6 号，我们在南开大学开展了首场 openEuler 创新实践课，14 号到 15 号，在兰州大学开展第二场课程。两所学校风格不同，但又有很多相似之处。

（一）

渤海之滨，巍巍南开，允公允能，日新月异。

![](https://mmbiz.qpic.cn/mmbiz_png/A0h5yD51CMZv61hU3O3Wc6w2ZJRtqJvtfLerNSTKDa7cBwoefFV7IXIkiaFlFHZOeLKIcG3Sq2fcsy40QibvqM3A/640?wx_fmt=png)

首场欧拉操作系统创新实践课在南开大学津南校区以一门面向全校本科生的选修课开展；两天课堂授课，两天线上辅导。共有 56 名同学参加，涵盖计算机类、智能科学、微电子等专业，当然还有 3 位选错课、可怜又可爱的文科生。

由于同学们均为一二年级的学生，大多之前从未接触过 Linux。二年级的同学刚上完操作系统课程还有一些基础，一年级的则是两眼一抹黑，这也为本次实践课的开展造成了不小的困难。

面对超过 2000 行的命令操作，同学们一度有点困惑和担忧，于是我们采取实验讲师王德祥在台上带领大家同步操作，我选择重要环节讲解实验原理和注意事项的形式，让同学们对实验的过程和涉及的理论有一定的理解，同时对部分产生畏难情绪的同学进行一对一辅导和鼓励。

![](https://mmbiz.qpic.cn/mmbiz_png/A0h5yD51CMZv61hU3O3Wc6w2ZJRtqJvtPSQCKicqXqBgjibL1ANGltq3VJVibVhP0VdBRWEqxwHgX3XLJcNcGS1dw/640?wx_fmt=png)

如果说同学们第一天的状况让我们三位讲师对大家能否完成工作量较大的实验忧心忡忡，那第二天大家的表现则只能用“开挂”来形容，好像人人心里都憋着一口气一定要做出结果给讲师、给自己一个交代。

我甚至把第一天上课的过程在脑子里复盘了一遍，确定自己没有“刺激”大家，更没有“威逼利诱”。有人放弃午休赶进度，有人为了完成当天的课程目标提前体验程序员的加班生活，一直到晚上 8 点左右终于完成实验。

> “第一天下午刚刚完成硬盘分区。晚上在宿舍想想，这一天下来，有那种被知识充满的感觉。紧接着，第二天上午，这将是一个印象深刻的上午，我先是迈过去了三个坑，紧接着就卡在第一遍 GCC 编译，一直编译不通过，直到中午 12 点多，还在编译，已经无暇顾及吃饭和午休。”
> 
> 下午一来，想想自己上午才做到 400 多行，而室友已经做到 2000 多行了，来不及想太多，便立刻投入“战斗”！1000 行……2000 行……3000 行,到晚上七点多时，在老师和周围同学的帮助下，终于是完成了操作系统的创建！
> 
> 上大学以来，很久没有在下课铃声响起后迟迟没有离开教室了；很久没有特别想要完成一个任务哪怕中午不吃饭、少点睡觉时间；而这些，在这两天里，却多次发生了在我身上。”
> 
> ---摘自 2020 级计算机专业 胡沈阳 《实践课总结》

也有人在讲师和学生双向“折磨”的 Linux 旅程中第一次感受到命令行的酷炫与魅力。

cd, scp，vim, make…，伴随着屏幕上一行行的输出滚动，“无情的复制粘贴机器”终于发现了通往新世界的大门。

![](https://mmbiz.qpic.cn/mmbiz_png/A0h5yD51CMZv61hU3O3Wc6w2ZJRtqJvtAu49Pl1WjJJMuk8c7QJbjnBJ8Qye0iccDW4DxichFNMpvsPTzO7fvoxg/640?wx_fmt=png)

在第二天的实验课程间隙，为了让大家能从紧张的操作中调整出来，我们临时安排 openEuler 社区运营总监马全一老师，结合多年的开源项目运营经验，和同学们分享了开源理念对大学生学习以及职业规划的积极影响。而这临时的安排，却成为很多同学参加本次课程的重要收获的一部分。

> “在 Linux 操作系统之外，我还了解了开源社区和开源这种思想的内涵。这对我的震撼是更大的。之前我对于计算机相关技术的认知和传统的知识是区别不大的。但是开源社区这一点深刻地改变了我的认知，每个人所写的代码都是开源的，任何其他人都可以学习，使用和分享。
> 
> 乍看之下这是一个非常愚蠢的想法，我自己写的东西都开源了，那我怎么赚钱呢？但实际上开源创造的价值要远远超出封锁所带来的，并且对整个技术的推进是有着重要价值的，并且国家也在十四五中强调开源。
> 
> 这上面的一切都让我从一个对开源一无所知的小白变成对开源充满兴趣的人。因此，我也因为这次的学习对开源社区获得了更多的知识，更加希望去开源社区提升自己。”
> 
> ---（摘自 2019 级智能科学与技术专业-卢帅琦 实践课总结）

能做到对新事物学有所思，并结合到国家政策方向，再立志投入这个方向去提升自己，在南开学子的身上，我看到了南开大学“允公允能，日新月异”价值观最生动的体现。

（二）

“偏一方而思四海，独一隅而雄九州”，短短十四字，却是属于每一个兰大人的自豪。欧拉操作系统创新实践课的第二站，我们来到了“联络四域，襟带万里”的西北重镇、河西走廊起点，兰州市。

此次创新实践课在兰州大学信息科学与工程学院的“2021 暑期实习实践”活动框架内开展，面向本部校区合作办学班的 230 余名本科生。欧拉开源社区，openGauss，MindSpore 三个开源社区分别交付为期两天的实践课程，一起构成兰大信息工程学院暑期实践活动的第一周课程内容。

![](https://mmbiz.qpic.cn/mmbiz_png/A0h5yD51CMZv61hU3O3Wc6w2ZJRtqJvtMEutas7PFZ99qiatVVIdCRvEkxXhLjC9uIZaqc5fqBt5WlM5YJw4azg/640?wx_fmt=png)

欧拉操作系统创新实践课第一天上午的理论课在兰大医学校区杏林楼大教室开展，我介绍 Linux 操作系统基础知识及欧拉开源社区，社区运营总监马全一向同学们介绍开源文化并获得了粉丝们在课件休息时的围观。接下来的动手实践在信息工程学院飞云楼 4 个实验教室开展，由实践课实验方案开发者赵小虎讲师讲解并带领操作。由于参加的学生人数较多，在开课前我们委托本次暑期活动学院负责人刘老师提前选拔 8 名基础扎实、动手能力较强的一/二年级助教，协助华为方讲师处理同学们在课程中遇到的问题。

![](https://mmbiz.qpic.cn/mmbiz_png/A0h5yD51CMZv61hU3O3Wc6w2ZJRtqJvtTD17vhymmFNPC69MxRsV9M0PMsbJBU6hliaIzZWX1KbYDVJDRsMxqicA/640?wx_fmt=png)

由于同学们均为大一/大二的学生，所以存在和在南开大学授课时类似的问题。但是得益于同学们的踏实认真以及助教的积极协助，绝大数同学还是坚持完成了实验并提交了作业。

（三）

7 月份开展的首批两场欧拉操作系统创新实践课，覆盖南开及兰大约 290 名本科生，共计 243 位同学向欧拉开源社区提交了他们的第一个 PR。

通过参加此次课程，同学们了解了 Linux 内核社区及其发行版、了解了欧拉开源社区及开源文化，并通过注册 Gitee, 签署 CLA，向社区提交 PR，熟悉了参与开源社区的基本流程；对于一些从未接触 Linux 的同学来讲，打开了有别于 Windows 界面的命令行操作新世界。

同时通过实地深入和同学们接触，我们也发现了一些早已投入开源社区贡献的学生，如兰大的华为开源俱乐部，以及主动维护 openEuler-Lanzhou-University 镜像站点的同学。

在这两场活动中，也暴露出课程设计中存在的问题，但同时让我们也找到了 欧拉开源社区在高校 Linux 操作系统教学中的用武之地。

目前开发的欧拉操作系统内创新实践课实验，没有考虑到高校大一大二学生不熟悉 Linux 操作系统使用的现状，造成课程效果并没有完全达到预期。另外，实验需要在笔记本上安装欧拉操作系统的虚拟机，而这个过程由于大家的电脑配置不同，安装的软件环境也不同，一定程度上耗费了不少时间。

接下来我们考虑优化课程，如尝试将实验移植到华为云上，减少同学们准备开发环境的时间；同时进行课程分级，针对大一/大二，大三/大四, 研究生的不同情况开发设计比较有针对性的课程。

另外，通过和高校老师及学生交流，我们发现国内很多高校取消了原本注重实践应用的《Linux 技术》课程，一定程度上造成计算机类学生在大三之前还未使用过 Linux 开发环境的问题。在上课的过程中我们发现之前接触过 Linux 的学生，在实验环节进展比较顺利，而且有一定的分析解决问题的能力。那么 openEuler 创新实践课以及社区的比赛、实习项目能否承担起解决高校计算机专业学生动手能力不足的问题？这对于 openEuler 来讲，仍然有很多工作要做。
