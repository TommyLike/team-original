GreatSQL 社区是万里数据库主导成立的数据库开源社区，是国内 MySQL 技术路线中最活跃的开源社区之一。

在刚刚结束的 openEuler Developer Day 2022 DB SIG 组工作会议上，GreatSQL 社区与 DB SIG 组内部达成共识并向欧拉开源社区技术委员会汇报。决定将 GreatSQL 加入 openEuler 22.09 软件仓库，GreatSQL 成为欧拉开源操作系统支持的首个 MySQL 系国产开源数据库。GreatSQL 将为社区开发者和用户提供优秀可持续的开源数据库技术能力，进一步繁荣欧拉开源操作系统的数据库软件生态，推动操作系统与数据库融合发展。

GreatSQL 社区目前已经成为 DB SIG 一员，未来 GreatSQL 社区将发挥开源数据库技术积累，打造基于 openEuler+多样性计算+GreatSQL 的底层核心技术底座，基于 GreatSQL 社区版本上大幅增强在欧拉开源操作系统上的性能。

#### 关于 GreatSQL

GreatSQL 社区成立于 2021 年，是一个 MySQL 开源数据库社区，由万里数据库发起，致力于通过开放的社区合作，构建国内自主 MySQL 版本及开源数据库技术，推动中国开源数据库及应用生态繁荣发展。

愿景：成为中国广受欢迎的开源数据库

网址：https://gitee.com/GreatSQL

GreatSQL 开源数据库专注于提升 MGR 可靠性及性能，支持 InnoDB 并行查询等特性，是适用于金融级应用的国内自主 MySQL 版本，可以作为 MySQL 或 Percona Server 的可选替换，用于线上生产环境，且完全免费并兼容 MySQL 或 Percona Server。

相较 MySQL 和 Percona Server，GreatSQL 进一步提升了 MGR 的性能及可靠性，并修复了多个相关 bug。此外，GreatSQL 还合并了由华为鲲鹏计算团队贡献的两个 Patch，分别针对 OLTP 和 OLAP 两种业务场景，尤其是 InnoDB 并行查询特性，TPC-H 测试中平均提升聚合分析型 SQL 性能 15 倍，最高提升 40 多倍，特别适用于周期性数据汇总报表之类的 SAP、财务统计等业务。

面向数据可靠性和架构的容错性要求更高，对多数据中心甚至多活有较高需求的金融级应用场景，GreatSQL 还新增了地理标签功能，采用全新的流控机制，防止性能抖动频繁出现，并提升同城双机房和跨城架构部署的可靠性。目前，GreatSQL 不仅可以作为 MySQL 或 Percona Server 的可选替代方案，用于线上生产环境，且完全免费并兼容 MySQL 或 Percona Server。

社区上游可用的分支版本有：GreatSQL 8.0.25、GreatSQL 5.7.36，能够完美兼容 MySQL 和 Percona Server。
